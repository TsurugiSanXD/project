1) ¿Que es un sistema de control de versiones?
R\: Es un sofware que ayuda a evitar la necesidad de guardar los archivos con nombres diferentes a medida que se realizan cambios en el archivo

2)¿Que diferencia hay entre un repositorio local y remoto?
R\: Que el retorio local se encuentra en el computador, mientras que el remoto se encuetra en el internet.

3)Mencione tres diferencias entre Git y GitHub.
R\: * Git es una herramienta usada para el control de veriones de manera local, mientras que GitHub por otro lado es una herramienta para la colaboración y el alojamiento remoto.
    * Git se utiliza a traves de linea de comandos, mientras que GitHub proporciona una interfaz web que facilita la visualización y gestión de repositorios Git alojados en su plataforma.
    * Git es un sistema de control de versiones distribuido y GitHub es una plataforma de alojamiento para repositorios Git.

4)Área de trabajo, área de preparación(ubicarse para la foto) y el directorio git(toma la foto).
  Podrías explicar en que consiste?
  R\: Área de trabajo: El área de trabajo es el directorio de tu sistema de archivos local donde trabajamos y contiene los archivos del proyecto.
      Área de preparación: La área de preparación es un espacio intermedio entre el área de trabajo y el repositorio Git. Nos permite seleccionar qué cambios específicos (archivos modificados) deseamos incluir en nuestra próxima confirmación (commit) al repositorio.
      Directorio git:  es donde Git almacena la metadata y la base de datos de objetos de tu proyecto. Esta es la parte central de Git donde se guarda todo el historial de versiones y la información sobre los cambios realizados en los archivos.

5)¿Para que se utilizan las ramas en Git?
R\: En git utilizamos las ramas para guardas diferentes versiones de un archivo.
